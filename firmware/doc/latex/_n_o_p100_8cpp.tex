\doxysubsection{Detailed Description}
Extensible firmware based on the NMEA2000 library. 

\begin{DoxyAuthor}{Author}

\end{DoxyCopyright}
This firmware is targetted at hardware based on the \href{https://github.com/preeve9534/NOP100}{\texttt{ NOP100}} module design. It implements a functional NMEA 2000 device that performs no real-\/world task, but which can be easily extended or specialised into a variant that can perform most things required of an NMEA 2000 module.

The firmware implements basic bus connectivity and the normal housekeeping required by NMEA through use of the \href{https://github.com/ttlappalainen/NMEA2000}{\texttt{ NMEA2000}} library. It also relieves any derived application of much of the hard work associated with module configuration using the switch inputs of the NOP100 hardware.

Handle a change of state on the PRG button. 

This function is called from loop() each time a state change is detected on the PRG button. A press of the PRG button simply starts a timer whilst a release triggers a callout (see below) which has responsibility for handling the interaction. The timer value allows a determination to made of whether the trigger was a short or long press of PRG.

In normal operating mode the Module\+Configuration interact() method is called to manage a configuration update protocol. In extended operating mode the extended\+Interact() function is called to manage special function protocols.

The return value from either of these callouts is used to set the status of the transmit LED or to switch firmware operating mode.
